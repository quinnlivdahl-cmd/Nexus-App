# React Dashboard Example: Review Surface

Use this lane when Codex needs to make a polished, inspectable web surface.

## Starter Shape

Target home:

`artifacts/mockup-sandbox`

Recommended ingredients already available in the sandbox:

- Vite + React
- shadcn/ui-style components
- Recharts
- Lucide icons
- Tailwind utilities

## Minimal Example

A useful first review surface should include:

- a compact header with artifact name, status, and last validation time;
- three to five summary metrics;
- a filterable table or card list of review items;
- one chart only if it answers a real review question;
- an evidence panel with exact file paths, commands, and issue links.

## Good Nexus Uses

- issue closeout dashboard;
- source-pack review surface;
- validation report browser;
- playtest state review;
- handoff packet browser.

## Avoid

- marketing hero pages;
- decorative mockups that do not help review;
- copying the live Nexus companion app layout unless the task is app UI work.
