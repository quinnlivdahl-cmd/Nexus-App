# Rich/Textual Example: Issue Console

Use this lane when the user needs a terminal-native review or operator surface.

## Minimal Panels

- queue summary;
- selected issue detail;
- validation command results;
- next action log.

## Good Uses

- issue queue browser;
- local validation dashboard;
- source index status;
- automation dry-run console.

## First Pass Rule

Use Rich for formatted command output first. Use Textual only when keyboard navigation, panes, or live refresh matter.
