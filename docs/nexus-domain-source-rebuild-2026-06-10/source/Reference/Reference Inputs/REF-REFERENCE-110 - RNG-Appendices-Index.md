# Nexus RNG Appendices Index
Revision: 1.0  
Status: Draft  
Last Updated: 2026-05-02

## Purpose

This index defines the RNG and balance appendix set for Nexus. These appendices support future decisions about core resolution, character creation, combat, resources, archetype viability, enemy design, content expansion, and eventual digital implementation.

These documents are reference and testing tools. They do not define official Nexus mechanics unless a later Active or Draft Active rules document explicitly adopts a mechanic.

## Canon Control Rule

The appendices may contain analysis, formulas, test models, candidate systems, and advisory design proposals. None of that material is source of truth by itself.

A mechanic becomes source of truth only when it is approved into the relevant Active or Draft Active project document, such as:

- Core Rules
- Combat System
- Player Character and Backgrounds
- Recruit NPC System
- Class Skill and Feat System
- Enemy and AI System
- Progression Economy and Resources
- Data Model and Content Schemas

## Evidence Labels

Appendix material should use these labels when helpful:

| Label | Meaning |
|---|---|
| Established | Mathematical fact, directly derivable result, or stable external reference. |
| Empirical | Result from simulation or structured testing under stated assumptions. |
| Advisory | Recommendation, interpretation, candidate design, or proposal. Not canon. |

## Recommended Use During Design

Use these appendices whenever a proposed mechanic changes:

- probability of success
- damage or effect output
- action economy
- defensive durability
- resource economy
- reroll or mitigation access
- character archetype identity
- enemy resistance profile
- encounter structure
- content scaling
- long-term progression

## Update Triggers

These appendices should evolve when design complexity increases. They do not need revision for every brainstormed idea.

Revise the relevant appendix when:

1. A core RNG candidate is added, rejected, or approved.
2. Attributes, skills, classes, backgrounds, origins, gear, recruits, or resources affect resolution math.
3. A new major archetype or playstyle is introduced.
4. A new enemy defense profile changes what builds are viable.
5. A new resource system changes luck control, action economy, or attrition.
6. Simulations reveal a dominant strategy or dead option.
7. A previously exploratory mechanic is promoted into a Draft Active or Active rules document.
8. A future Replit implementation needs reproducible random-number behavior.

## Relationship to the Main RNG Research Report

The main RNG research report should remain relatively stable. It is the library.

These appendices are the laboratory. They can be updated repeatedly as Nexus becomes more concrete.
