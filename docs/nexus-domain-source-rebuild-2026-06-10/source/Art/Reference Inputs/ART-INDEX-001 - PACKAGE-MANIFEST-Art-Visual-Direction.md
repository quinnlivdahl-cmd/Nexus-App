## Evidence Preserved

Art transition evidence includes:

- visual style guidance;
- TacMap collage and image workflow handoffs;
- SVG starter kit references;
- icon grammar and node-edge marker notes;
- session/reference images;
- old asset register material;
- Art dashboard state and handoffs.

This evidence informs Art but does not automatically make an asset canon.

## Handling Decisions

- Markdown visual direction and indexes belong in Art.
- Large binary images stay in asset folders or tool-specific storage.
- Session images remain session/reference assets until promoted.
- Prompt library is seeded, not final.
- Asset canon status and routing live in `ART-RUNBOOK-008`.
- TacMap visual workflows must stay aligned with Combat, Play Aids, Automation, and Dashboards.

## Cross-Domain Routing

- `Combat`: TacMap rules, tactical movement, cover, hazards, and combat timing.
- `Play Aids`: player/DM-facing display aids and printable/table surfaces.
- `Automation`: renderers, exports, runtime schemas, and companion-tool behavior.
- `Dashboards`: live asset work status and current art tasks.
- `Reference`: inspiration, external comparison, and non-canon research.
- `Data`: structured asset metadata, indexes, and import/export tables.

## Preservation Rule

No older asset, handoff, dashboard, image, SVG kit, prompt set, or prior art source file is deletion-approved by this manifest.

Cleanup requires asset coverage verification, source coverage verification, local storage confirmation, and explicit user approval.
