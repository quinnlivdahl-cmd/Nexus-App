# Play Aids Domain Readme

## Purpose

The `Play Aids` domain owns route-node maps, TacMap display aids, player-facing handouts, printable/display exports, external table companion concepts, and related display-spec guidance.

Play Aids answers: **how should play be shown, shared, printed, recapped, or displayed without turning the aid into the rules engine?**

## Authority Boundary

Play aids make play easier to see and use.

Play Aids does not own:

- core game loop or mission-node structure;
- combat rules, movement, cover, statuses, hazards, or encounter pacing;
- automation architecture, renderer implementation, or runtime/export truth;
- live dashboards, task tracking, or current campaign state;
- visual style, SVG asset style, prompt patterns, or asset indexing;
- raw SVG/PNG/design assets.

## Current Structure

Current Play Aids surfaces include:

- `PLAYAID-INDEX-000`: domain readme.
- `PLAYAID-INDEX-001`: domain manifest.
- `PLAYAID-OVERVIEW-001`: auxiliary play aids overview.
- `PLAYAID-ROUTE-001`: route-node map companion.
- `PLAYAID-MAP-001`: solar system map and faction display.
- `PLAYAID-TACMAP-001`: TacMap display aids.
- `PLAYAID-STATE-006`: player-facing dashboards and handouts.
- `PLAYAID-PRINT-001`: Canva, PDF, and printable aids.
- `PLAYAID-DISPLAY-001`: external display and table companion concepts.
- `PLAYAID-OPEN-009`: auxiliary play aids open questions.

## Cross-Domain Routing

- `Core`: route-node structure, mission-node structure, and route-node end report requirements.
- `Combat`: combat rules, TacMap movement, cover, statuses, hazards, and encounter pacing.
- `Automation`: renderer, export/import, runtime packet, and companion-tool implications.
- `Dashboards`: live state and current trackers.
- `Art`: visual direction, icon style, SVG style, and image/collage direction.
- `Data`: structured truth tables and exportable field definitions.

Play Aids can specify the display surface, but the owning domain defines the rule or meaning.

## Current Maturity

Play Aids is a working draft.

It supports encounter-start packets, route-node end reports, Obsidian-first handouts, TacMap display specs, layered collage workflows, repairability, player-safe/DM-only layer handling, and fallback display principles.

## Preservation Rule

No prior play aid, SVG kit, dashboard, handoff, map prototype, route-node display note, generated visual reference, or output-register fragment is deletion-approved by this readme.
