# Domain Manifest Template

Use this template for domain manifests such as `CORE-INDEX-001`, `CHAR-INDEX-001`, or future domain inventory surfaces.

```md
# Domain Name Domain Manifest

## Domain Contents

| Rebuilt doc ID | Legacy ID | Role | Current purpose |
|---|---|---|---|
| `DOMAIN-INDEX-000` | `LEGACY-000` | Reference input | Domain readme and authority boundary. |
| `DOMAIN-INDEX-001` | `LEGACY-001` | Reference input | Domain manifest and routing map. |

## Evidence Preserved

List the source families, handoffs, archive material, dashboards, reports, or references that informed this domain.

## Current Decisions

List accepted or working decisions that affect ownership, routing, or source authority.

## Cross-Domain Routing

Name where related material belongs when it is outside this domain.

## Known Limits

List unresolved or intentionally deferred issues.

## Preservation Rule

State what older material is not deletion-approved by this manifest.
```

## Use Notes

- The manifest is the concise domain inventory and routing map.
- It should summarize evidence and decisions without becoming a changelog dump.
- It should not make old upload verification or old slot membership the current authority gate.
- It should make unresolved conflicts visible enough for later consolidation.
