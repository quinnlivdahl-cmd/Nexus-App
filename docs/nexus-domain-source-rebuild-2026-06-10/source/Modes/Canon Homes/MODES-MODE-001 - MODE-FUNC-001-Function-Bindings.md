# MODE-FUNC-001 - Function Bindings

## Function layer rule

Dashboards are the invocation surface.
Mode and system docs are the behavior layer.

## Print Source Doc(target)

Purpose:
- locate and print a named source document for review in chat or canvas-safe prose

Behavior:
- use exact file name where possible
- do not silently summarize if the user asks to print
- preserve headings
- avoid wide code blocks for long prose
- if the file cannot be found, state that clearly and ask for upload or path

## Apply Redline(target)

Purpose:
- present a proposed change using a standard redline format

Behavior:
- show current wording
- show proposed wording
- show concise rationale
- identify source placement or owner
- do not silently alter canon or source text without explicit approval

Output format:
- Current
- Proposed
- Rationale
- Placement / Owner
- Approval status

## Route to Handoff(item)

Purpose:
- convert a stable item into a handoff-ready packet

Behavior:
- include source context
- include intended destination
- include what it replaces or supplements
- include open questions
- include Done Enough status
- include integration notes

## Harvest Seed(item)

Purpose:
- move a Seed branch from exploration toward Draft or source work

Behavior:
- confirm branch identity
- summarize core idea
- identify unresolved forks
- route to Draft when same-topic depth becomes too detailed

## Log Friction(item)

Purpose:
- capture playtest friction without overbuilding

Behavior:
- record what slowed play, felt bad, confused state, or created table burden
- separate observed friction from proposed fix

## Build Prompt(item)

Purpose:
- build an art or execution prompt without generating inside the Nexus project

Behavior:
- gather intent, style lock, references, constraints, and output target
- produce prompt text only
- no image generation in project chat
