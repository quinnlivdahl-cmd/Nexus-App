# Nexus Obsidian Folder Map

## 1. Purpose

This document describes the Obsidian folder routing pattern that supports the legacy source slot map and the broader vault structure.

It does not require the full vault to be uploaded to external chat context. Obsidian remains the complete project database; targeted chat context uses only selected loose files and modular source bundles.

## 4. Placement Rules

- Use granular source files in Obsidian even when sources receive source bundles.
- Keep old versions and raw backups in Obsidian archive or original backup storage, not in chat-context bundles by default.
- Keep package READMEs/manifests near their package docs and mirrored in source bundles.
- Keep generated outputs recorded in the Output Register.
- Do not delete old files simply because a newer folder map exists.
