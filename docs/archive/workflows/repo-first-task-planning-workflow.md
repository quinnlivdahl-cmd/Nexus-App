# Repo-First Task Planning Workflow

Status: superseded workflow candidate; historical pointer only

This June 2026 candidate proposed a broad repo-first planning system. Planning Ownership #81 superseded its competing planning-state model.

Use the current single-owner model instead:

| Information | Owner |
|---|---|
| Executable task state and acceptance criteria | Live GitHub Issues |
| Durable human-facing plans, findings, and progress | Obsidian Nexus hub `02 Planning` |
| Temporary current-session sequencing | The active chat or tool plan |
| Cross-session executable continuity | A linked GitHub Issue comment or approved handoff |

Current stable workflow behavior is defined by:

- repo agent routing: `AGENTS.md`;
- issue intake and closeout: `docs/agents/issue-tracker.md`;
- session focus and planning ownership: `docs/admin/task-planning/codex-session-discipline-workflow.md`;
- independent review: `docs/admin/task-planning/nexus-review-rubric.md`;
- bridge and handoff routing: `docs/chatgpt-project-bridge/README.md`;
- maintained Obsidian location: `docs/admin/nexus-distributed-surfaces.md`.

The original candidate remains recoverable through Git history. Do not restore its repo planning overlay, duplicate task queue, proposed folder tree, or mandatory planning-file update rules.
