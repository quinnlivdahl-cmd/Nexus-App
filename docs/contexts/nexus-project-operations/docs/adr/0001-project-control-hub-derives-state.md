# Project Control Hub derives state from authoritative systems

Nexus Project Control is a private, owner-only projection and launch surface rather than a new authority. It derives ticket, roadmap, worktree, and planning context through timestamped Project Snapshots and a curated Nexus Context Bundle manifest: GitHub Issues remain the execution tracker, canonical source remains game truth, the repo remains implementation authority, Obsidian remains project memory, and Drive holds curated GPT-retrieval material. The hub prepares reviewable Codex Launch Packets and routes mutations through Codex with explicit user approval; it does not directly mutate GitHub or Drive in its first release.
