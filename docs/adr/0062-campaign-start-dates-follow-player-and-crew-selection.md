# Campaign start dates follow player and crew selection

In a Legacy World, the first call of the existing two-call Campaign Opening Pipeline proposes the exact campaign start date jointly with the Main Campaign Thread, initial Director State, and Node 0 selection after the Player Character and Starting Crew are known. Deterministic validation requires forward World Calendar chronology, compatibility with current Legacy Consequences, and continued viability and availability of the selected crew; an invalid date rejects the opening proposal for bounded repair rather than removing or replacing a selected character. This preserves lore chronology without adding a third opening call or turning calendar setup into a player chore.

ADR 0066 makes selected-character availability depend on deterministic Biological Age, Stasis, Bioform, and continuity rules rather than raw calendar difference or Director judgment.

ADR 0063 permits the same call to propose bounded Interval Developments, which are applied to a candidate world snapshot and validated before the campaign premise can commit.
