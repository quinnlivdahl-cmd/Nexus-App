# Characters Domain Readme

## Purpose

The `Characters` domain owns player-character and crew-facing source material: character assembly, chassis/body implications, origins/personhood-facing character rules, disciplines/roles, traits/feats/progression, crew rosters, recovery/revival consequences, and character open questions.

Characters answers: **what can a player character be, how do they grow, what happens to their body/personhood in play, and what consequences follow damage, recovery, crew state, and advancement?**

## Operating Principle

The active design principle remains:

```text
There are no fantasy races in Nexus. There are only arguments about what still counts as human.
```

Characters should be usable in DM Mode now, but exact math, final point costs, origin balance, and several cyberware/recovery questions remain open.

## Authority Boundary

Characters owns character-facing implications.

Characters does not own:

- cyberware item mechanics, costs, equipment implementation, or gear-side Firewall/System Status;
- broad lore, caste, personhood law, faction ideology, and splicer history beyond character-facing implications;
- combat action economy, status timing, or tactical display;
- Skills-side ability math, Target Score, or resolution rules;
- live campaign crew state unless abstracted into a reusable character-system rule.

## Key Preserved Decisions

- Handoffs are potential unincorporated evidence and must not be ignored.
- `Cyborg` is not a default separate starting chassis pick by itself.
- Cybernetic identity generally emerges from cyberware selected at character creation.
- Cyberware investment creates a derived integration tier.
- Full Chassis Cyborg remains distinct from Upload.
- Upload starting bodies are preserved as Base Sleeve, Maintenance Drone, and Combat Shell.
- Splice + cyber combinations are possible for PCs but rare, expensive, constrained, and drawback-heavy.
- Standard Fit is the character/infrastructure compatibility concept.
- Character durability uses Health, System Integrity, Defense, Firewall, Mitigation, and Shield as the current spine.

## Cross-Domain Routing

- `Equipment`: cyberware item trees, item mechanics, costs, tools, gear, and equipment-side defensive systems.
- `Lore`: splicer origin history, Designer Human culture, Vatborn caste, C-POH doctrine, personhood law, faction attitudes, and full setting arguments about what counts as human.
- `Combat`: tactical statuses, combat timing, cover, action economy, and combat display.
- `Skills`: ability taxonomy when resolution-facing, Target Score, skill hierarchy, and advancement math where skill-owned.
- `Dashboards`: active campaign crew state and current playtest state.

## Preservation Rule

No old character source, handoff, dashboard, backup, package, or recovery/cyberware draft is deletion-approved by this readme.
