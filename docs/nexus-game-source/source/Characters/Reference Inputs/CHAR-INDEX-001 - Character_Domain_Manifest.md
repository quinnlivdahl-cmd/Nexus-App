## Conflict Decisions

| Conflict | Current handling |
|---|---|
| Cyborg as race/chassis vs cyberware-derived identity | Use cyberware-derived integration tier for character creation. Preserve Cyborg as possible legal/cultural/faction/personhood label. |
| Full Chassis Cyborg vs Upload | Full Chassis Cyborg retains biological brain/person-core; Upload is digital/person-substrate embodiment. |
| Splice + cyber best-of-both-worlds risk | Allowed for PCs, but rare, expensive, constrained, and drawback-heavy. |
| Provisional chassis usefulness vs unfinished math | Keep chassis DM-usable through affordances/tradeoffs, not final stat blocks. |
| Active campaign crew state | Route to `Dashboards` unless abstracted into a reusable system rule. |
| Cyberware item trees | Character identity model here; cyberware item mechanics and costs route to `Equipment`. |

## Cross-Domain Routing

- `Admin`: governance rule that handoffs may contain unincorporated evidence.
- `Modes`: DM display and mode behavior.
- `Core`: pre-node advancement/loadout and recovery consequence hooks in the campaign loop.
- `Combat`: character-derived Firewall, hack-surface, durability display, and combat-state interactions.
- `Lore`: lore-heavy caste, personhood, faction, splicer, Standard Fit, and human-continuity material.
- `Equipment`: cyberware, EMP, tools, Shields, Mitigation, and item implementation.
- `Skills`: ability taxonomy and advancement math when resolution-facing.
