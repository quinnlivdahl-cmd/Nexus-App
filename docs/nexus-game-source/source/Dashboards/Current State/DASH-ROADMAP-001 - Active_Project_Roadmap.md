# Active Project Roadmap

## 1. Current operating priority

Nexus source files are the project brain. Active source files should shape substantive work before proposals, evaluations, recommendations, drafting, routing, Seed work, Art/TacMap work, DM/playtest rulings, source placement, dashboards, and implementation planning.

This is an operating behavior, not a visible output ritual.

## 2. Immediate roadmap

### 1. Complete source update and routing

Current status: in progress.

Installed / candidate updates:

- Global mode operations source-first routing update.
- Stewy read-first/source-routing update.
- Bootstrap and source-routing updates.
- ADM-013 source-first routing map.
- Handoff-content implementation work beginning with noncombat systems, ability taxonomy, TacMap workbook/build-plate process, Art TacMap workflow, and this dashboard.

Goal:

- make future Nexus work route through sources by default;
- reduce repeated re-litigation of settled material;
- make handoffs become durable working source where appropriate.

### 2. Return to Seed for level-up ability list

Goal:

- produce a small workable ability list for the test campaign level-up.

Current source substrate:

- Attribute -> Skill -> Skill Focus -> Ability taxonomy.
- Tier / Rank / Focus Total model.
- Provisional ability tree seed in SRC-CHAR-009.
- Non-combat scene working model in SRC-SKILL-009.

### 3. Create one working TacMap

Goal:

- create one playable TacMap for the test campaign.

Current source substrate:

- TacMap node movement and positioning source.
- TacMap workbook/build-plate process in SRC-COMBAT-009.
- TacMap art workflow in ART-012.

Priority:

- playable geometry;
- readable node/path/cover/elevation logic;
- overlay compatibility;
- enough art to test, not endless polish.

### 4. Return to DM/playtest

Goal:

- level up the test characters;
- test social systems;
- test combat systems;
- test the TacMap;
- continue the campaign/playtest loop.

Playtest should capture gaps, not block on final rules perfection.
