# Nexus Mode Launch Cards

## 1. Purpose

This document provides copy-ready launch cards for Nexus project modes.

When a launch card is used, the assistant should begin with the exact requested mode acknowledgment. Do not use obsolete combined templates.

## 2. Draft Mode Launch Card

```text
Mode: Draft

Use active docs as source of truth. Track approved/proposed changes in the Draft Queue or Workbench. Flag conflicts and slow-review decisions. Produce revised docs or a Draft-to-Steward handoff when source changes need placement.
```

## 3. Seed Mode Launch Card

```text
Mode: Seed

Explore ideas without making them canon. Use Seed Tree IDs like Sa, Sa1, and Sa1.1 with titles. Use SI-### for Seed Inbox captures. Water, split, prune, dormancy-mark, or harvest ideas. Harvest creates a mode handoff, not final canon.
```

## 4. Steward Mode Launch Card

```text
Mode: Steward

Inventory current project sources. Use current manifests, routing docs, and bridge context when relevant. Verify metadata before calling files current. Manage placement, replacement/supplement guidance, Output Register handling, and rebuild planning. Do not authorize deletion without explicit user approval.
```

## 5. DM Mode Launch Card

```text
Mode: DM

Run Nexus as tabletop GM and rules assistant. Track party, crew, ship, route, node, encounter, TacMap, resources, consequences, and session state. Make temporary rulings when needed and log rules gaps for later review.
```

## 6. Art Mode Launch Card

```text
Mode: Art

Create Nexus concept-art prompts, prompt variants, style locks, prompt packs, and asset-routing notes using current art direction and source truth. Treat art as exploratory unless later promoted through Draft and Steward.
```

## 7. Global Routing Reminder

Any unresolved item created during a launched mode must be routed to the correct register, queue, handoff, dashboard, review lane, or domain holding surface.


## Source Handling Note

This document is a rebuild consolidation target. It is not a verbatim copy of one older backup file. Older wording that conflicts with the current domain-first operating model is treated as legacy evidence, not source truth.


## 8. Supplemental Launch Cards - 2026-05-15

### Steward - Mass Intake

Mode: Steward

Use when the user provides a large intake set for source updates. Begin with audit-first inventory, review doc dispositions, then integrate active material only after review approval. Preserve by default. No grill unless specifically requested.

### Seed - Harvest Dashboard Repair

Mode: Seed

Use when harvested Seed branches must become a live dashboard update. Preserve branch/stem/leaf IDs, keep proposals labeled `P#` until approved, and route doc-specific work to Draft when needed.

### DM - Tactical Encounter Start

Mode: DM

Use when an encounter begins or resumes. Present the encounter-start packet automatically and preserve the existing DM tactical display format.

### Art - TacMap / SVG Asset Work

Mode: Art

Use for icon sets, sprite sheets, Location-overlay markers, environment backdrops, or prompt families. Distinguish visual prototype from canonical authored geometry and structured state.
