# Nexus RNG Appendix E: Content Budget and Power Budget
Revision: 1.0  
Status: Draft  
Last Updated: 2026-05-02

## Purpose

This appendix defines a draft framework for evaluating future Nexus content so expansions do not accidentally create dominant strategies, power creep, or dead options.

The goal is not to make every feature equal. The goal is to understand what each feature improves and what it costs.

## Canon Control

This appendix is an advisory balance tool. Its scoring concepts do not become official rules unless approved in the relevant Active or Draft Active project document.

The appendix may recommend that an ability is risky, underpriced, over-stacked, or too weak. Those recommendations are not binding until adopted.

## Core Principle

A mechanic becomes dangerous when it improves multiple power axes at the same time without enough cost, risk, restriction, or opportunity trade-off.

Example:

An ability that grants extra movement, extra accuracy, extra damage, and an extra action is not four small bonuses. It may be a multiplicative package.

## Power Axes

| Axis | What It Improves | Common Risk |
|---|---|---|
| Accuracy | Chance to land an effect. | Makes specialists automatic. |
| Damage | Harm per successful action. | Creates burst dominance. |
| Effect strength | Status, debuff, heal, shield, or utility value. | Can bypass damage balance. |
| Action economy | Number, timing, or quality of actions. | Usually multiplicative. |
| Defense | Survival, mitigation, evasion, resistance. | Can erase encounter pressure. |
| Mobility | Positioning, escape, reach, objective play. | Can invalidate terrain and enemies. |
| Range | Ability to affect targets safely. | Can erase melee risk. |
| Reliability | Rerolls, advantage, guaranteed minimums. | Reduces meaningful failure. |
| Resource efficiency | More output per cost. | Creates long-term dominance. |
| Scaling | Improves as stats/content expand. | Causes future power creep. |
| Synergy | Multiplies other features. | Hard to evaluate in isolation. |
| Information | Scouting, prediction, hidden-state control. | Can solve encounters before they happen. |

## Power-Creep Warning Flags

Flag a feature for review if it:

1. Adds action economy.
2. Adds both accuracy and damage.
3. Adds reliability and burst output.
4. Makes a weakness matchup neutral.
5. Scales with number of attacks, summons, allies, or triggers.
6. Triggers on every hit, every crit, every kill, or every round.
7. Reduces cost of a resource loop.
8. Improves offense and defense at the same time.
9. Has no meaningful counter-profile.
10. Becomes stronger as more content is added.

## Draft Cost Categories

Costs may include:

| Cost Type | Example | Risk |
|---|---|---|
| Action cost | Spend main action. | Fair but may feel bad if effect fails. |
| Bonus/minor action cost | Spend smaller action. | Easy to underprice. |
| Resource cost | Spend focus, heat, ammo, charge, etc. | Depends on refresh rate. |
| Positional cost | Must be close, exposed, flanked, stationary. | Strong if enemies can punish it. |
| Time cost | Setup turn, cooldown, delayed effect. | Weak if fight is too short. |
| Risk cost | Backfire, stress, heat, self-damage. | Must be real, not cosmetic. |
| Opportunity cost | Cannot use another strong option. | Often the cleanest limiter. |
| Build cost | Requires trait, class, gear, or specialization. | Can become tax if mandatory. |

## Multiplicative Stack Rule

Advisory rule:

When a feature improves more than one of the following at once, treat it as high-risk until tested:

- chance to succeed
- number of attempts
- effect size
- target count
- action count
- resource efficiency
- survivability

## Content Expansion Test

Before adding a new content family, ask:

1. Does it create a new strategy or merely improve an existing best strategy?
2. Does it open a niche for a weak archetype?
3. Does it stack with already strong features?
4. Does it require a new enemy counter-profile?
5. Does it make old content obsolete?
6. Does it require a new simulation test?

## Update Triggers

Update this appendix when:

- a new class, ability tree, gear type, origin, or recruit trait is proposed
- a resource cost is defined or changed
- a bonus type becomes common
- action economy rules change
- simulations reveal power creep
- a feature becomes mandatory or irrelevant in playtests
