# Reference Domain Readme

## Purpose

The `Reference` domain preserves non-authoritative research, surveys, system comparisons, RNG appendices, lore guidance, seed/reference material, PDF indexes, and research reports.

Reference answers: **what evidence, rationale, comparison material, or research context might inform source work?**

Reference material is not canon by default. It becomes source only when promoted through the source workflow into the appropriate domain canon home, applied rule, deferred seed, or open question.

## Authority Boundary

Reference owns evidence and research context.

Reference does not own:

- current rules;
- lore canon;
- campaign state;
- mode operating behavior;
- final data values;
- asset canon;
- deletion/supersession decisions.

When reference material conflicts with active source, preserve the reference as evidence and route the conflict to the appropriate domain or review lane.

## Current Structure

Current Reference surfaces include:

- `REF-INDEX-000`: domain readme.
- `REF-INDEX-001`: domain manifest.
- `REF-SURVEY-*`, `REF-INSPIRATION-*`, `REF-RNG-*`, and `REF-SCIFI-SEED-*`: durable reference summaries, comparison reports, and research seeds.
- `REF-REFERENCE-*`: research reports, appendices, PDF indexes, and source-material indexes.
- `REF-RUNBOOK-*`: promotion or application rules that help decide whether evidence should move elsewhere.

The folder structure may be refined later, but the core rule is stable: reference material supports source decisions without replacing source.

## Promotion Routes

Route promoted reference material by function:

- mechanics evidence to `Core`, `Combat`, `Skills`, `Equipment`, or `Characters`;
- lore evidence to `Lore`;
- content ideas to `Content`;
- art or visual evidence to `Art`;
- structured numeric/table evidence to `Data`;
- automation/runtime research to `Automation`;
- mode behavior evidence to `Modes`;
- unresolved alternatives to the target domain `Open Questions`;
- incomplete future drafting material to the target domain `Deferred Seeds`.

Promotion should include a short rationale and, when useful, the reference path used.

## Obsidian and Archive Handling

Reference files remain useful in Obsidian because the user may browse research, PDFs, reports, and old support material outside Codex.

Do not delete old reference folders, source materials, PDFs, RNG appendices, research reports, or backups merely because a domain readme exists. Cleanup requires coverage verification and explicit approval.
