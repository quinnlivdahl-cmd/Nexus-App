# Canva PDF and Printable Aids

## 1. Purpose

Printable and design-platform aids turn source guidance into table-ready or player-facing artifacts. They can include PDFs, quick-reference cards, visual handouts, map sheets, and presentation-style aids.

## 2. Good candidates

Good candidates include:

- compact combat card;
- status icon reference;
- TacMap legend;
- derived Location/state snapshot;
- route-node end report sheet;
- player-safe faction/route handout;
- equipment/loadout card;
- condition/status reference;
- printable objective card.

## 3. Placement

- Markdown source meaning remains in source packages.
- Raw editable design files and images should go to Obsidian asset/design folders.
- Finished PDFs/PNGs should be archived or linked as exports, not treated as the only source.
- `Art` owns art direction and reusable visual style.
- `Play Aids` owns display-aid guidance.

## 4. SVG/PNG/PDF export routing

SVG icon kits, sprite sheets, PNG previews, and printable layouts are valuable assets, but they should not displace the source docs that explain what the icons mean.

Routing:

- SVG starter kit and sprite sheet: assets plus `Art` visual-direction reference.
- TacMap legend or player-facing icon guide: `Play Aids` display aid.
- Renderer-ready asset package: `Automation` and `Art` bridge, not source doctrine by itself.
- Printable map or session handout: `Play Aids` aid plus campaign archive if used.

## 5. Source safety checklist

Before producing polished design exports, verify:

- the source meaning is stable enough for the artifact;
- player-facing and DM-only information are separated;
- the artifact states snapshot/currentness when needed;
- source docs remain editable and preserved;
- the export does not imply final canon beyond approved source.

## 6. When to use design tools

Use design tools when layout, art, or printability matters. Do not use them when a Markdown list would be faster and more maintainable.

## 7. Printable constraints

Printable aids should:

- remain legible at intended size;
- avoid tiny icons without a legend;
- keep tactical information clear;
- work in grayscale or low-ink versions when possible;
- include source/version notes when used as durable handouts.

## 8. Canon/non-canon routing

A polished visual is still non-canon unless its content is approved and reflected in the appropriate source/state doc. Preserve visually useful drafts, but do not let polish outrun source status.
