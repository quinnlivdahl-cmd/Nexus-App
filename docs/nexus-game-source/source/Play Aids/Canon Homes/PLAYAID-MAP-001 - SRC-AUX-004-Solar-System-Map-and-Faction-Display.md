# Solar System Map and Faction Display

## 1. Purpose

Solar-system maps and faction displays help players understand where the crew is, what routes are available, who controls what, and why route choices matter.

## 2. Current desired coverage

Potential display layers:

- current ship location;
- available travel routes;
- faction zones of influence;
- station/habitat/mine categories;
- known danger or opportunity nodes;
- known route clocks;
- crew relationships or faction access;
- major hidden/unknown areas, if player-facing.

## 3. Stability level

This category remains early. Do not overcommit to final solar-system layout, faction borders, or named locations until `Lore` and current campaign state are more mature.

The uploaded orbital/station visual reference should be preserved as visual evidence/reference, but not promoted as a named canon location or final system map.

## 4. Faction display guidance

Faction displays should support play, not become lore encyclopedias.

Useful fields:

- faction name or working label;
- visible attitude/status;
- access or restriction;
- known locations;
- current pressure/clock;
- player-facing rumor/lead;
- DM-only hidden state if applicable.

`Lore` owns faction and world truth. `Dashboards` owns current faction state. `Play Aids` owns display format guidance.

## 5. Station infrastructure overlays

Future station/habitat maps may display practical infrastructure themes such as:

- Standard Fit assumptions;
- suit/clinic/dock accessibility;
- faction-controlled zones;
- transit routes;
- cargo/industrial/crew/public spaces;
- splicer or nonstandard-body accommodation disputes;
- emergency systems and hazard zones.

These are optional overlays, not mandatory map layers.

## 6. Map formats

Supported possibilities:

- Markdown region list;
- simple route graph;
- Obsidian Canvas;
- static image reference;
- printable player map;
- future external companion display.

## 7. Source safety rules

Do not promote an image into canon solely because it looks good. Images can inspire later names, places, or layouts, but lore/campaign state must be explicitly approved or recorded in the appropriate source/state docs.
