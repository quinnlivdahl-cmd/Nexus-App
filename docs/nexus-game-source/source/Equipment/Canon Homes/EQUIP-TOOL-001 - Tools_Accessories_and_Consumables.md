# Tools Accessories and Consumables

## 1. Tool slot

The Tool slot is a major build/loadout anchor.

A tool is not automatically charge-based. Depending on the specific item, it may provide:

- actions;
- permissions;
- modifiers;
- tags;
- access;
- passive abilities;
- triggered abilities;
- cooldowns;
- charges;
- deployment of a drone, device, kit, sensor, hacking route, med tool, or field system.

## 2. Tool examples as source-supported categories

Do not treat this as a final catalog. Current source supports examples such as:

- hacking deck;
- breaching charge;
- medkit or medical tool;
- drone controller or drone item;
- scanner;
- repair kit;
- forge/credential kit;
- sensor link;
- remote device;
- body/chassis-enabled tool route.

## 3. Accessories

Accessories replace the old generic small-pack model by default.

An accessory slot may hold:

- one discrete item; or
- one small themed kit.

Source-preserved example:

```text
Forgery Kit = 1 Accessory
```

Accessories should ideally have both combat and noncombat use cases when possible, but they do not need equal value in both modes.

## 4. Consumables

Consumables are explicitly counted and restored only through:

- inventory;
- resupply;
- fabrication;
- market access;
- mission reward;
- scenario permission.

Current direction: capped consumables exist, but exact cap model remains unresolved.

Possible working model for DM play:

```text
Character has capped consumable slots.
Some items may occupy accessory slots instead.
Powers, gear, body traits, or ship systems may modify caps.
```

This is a DM-usable placeholder, not final balance.

## 5. Remote and micro interactions

Remote micro-interact requires permission from:

- tool;
- drone;
- hack;
- sensor link;
- remote limb;
- pre-placed device;
- authorized system route;
- body/chassis capability;
- signal/contact effect.

This keeps remote interaction tied to equipment, skills, body, or fiction.

## 6. Charges, cooldowns, and readiness

Do not assume every tool has charges.

Use charges/cooldowns/readiness only when the item is meant to create tactical choice, resupply pressure, or scenario risk.

Good reasons to track a limit:

- scarce materials;
- mission-specific explosive/breach charge;
- heat or battery pressure;
- legal/faction tracking;
- hack trace;
- one-shot preparation;
- dramatic resource scarcity;
- powerful effect that needs a limit.

## 7. Mass-intake clarifications

### 8.1 Tools as permission surfaces

Tools may provide:

- permissions;
- actions;
- modifiers;
- tags;
- access;
- remote interaction;
- deployment of drones, devices, kits, sensors, or field systems.

A tool does not need charges by default. Charges, cooldowns, supplies, or breakage should appear only when they create useful tension or clarify limited use.

### 8.2 Remote Micro-Interact permission

Remote Micro-Interact requires permission from at least one valid source, such as:

- tool;
- drone;
- hack;
- sensor link;
- remote limb;
- pre-placed device;
- authorized route;
- body/chassis capability;
- Signal/Contact effect.

### 8.3 Assist linkage

Basic Assist can remain simple. Strong, ranged, reliable, repeated, multi-ally, or scene-altering Assist usually requires a build feature, gear, power, tool, drone, or scenario permission.

### 8.4 Consumables and pickups

Consumables remain capped by their own rules. Pickups do not become free active consumables unless a scene says they can be used immediately or they are equipped during a valid loadout/preparation window.
