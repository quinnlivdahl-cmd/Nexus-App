# Naming, Language, and Cultural Style

## Status

This is a seeded guidance doc, not a completed naming bible.

Memory Overflow preserved a naming caution: avoid accidental repeated or overly similar names. This package also preserves placeholder-name status for major lore handles.

## Placeholder terms

The following are not final names unless later approved:

- `C-POH`
- `UN`
- `Treaty Order`
- `System Authority`
- `Collective`
- some campaign-derived entity labels
- `Designer Human`
- `Vatborn`
- `Low-G / Adapted Human`
- `Splicer` as umbrella term

## Choir / Collective naming rule

Use **Choir** as the preferred working concept.

Use **Collective** only as a placeholder, outsider shorthand, public misdiagnosis, legacy label, or mask. It is not the official name of a clean organized faction by default.

Preferred phrasing:

- "Choir-affected conglomeration"
- "Choir-affected institution"
- "implant-linked organization"
- "mind-implanted people affected by the Choir"
- "the so-called Collective" when speaking from an outsider or public perspective

Avoid presenting `the Collective` as a tidy polity unless a later Draft/Seed pass changes that.

## Style principles

Nexus names should feel practical, institutional, technical, legal, corporate, habitat-born, or culturally worn-in. Avoid fantasy-race naming unless a culture explicitly justifies it.

Good naming families to develop later:

- treaty and legal institutions;
- corporate habitat names;
- station nicknames;
- ship names;
- upload/server polity naming;
- C-POH internal terms;
- splicer/vatborn slang;
- cyberware clinic and repair-rights terms;
- Signal/Choir public rumors and coded language.

## Campaign-name caution

Names from active campaigns should not be globally reused without checking campaign state. Heliomed, E43, Nereid-3, Ternary Lock, Rill, Voss, C-WARDEN, and Special Review Detachment are campaign-derived names until promoted.
